# mephi-session-project-2026

## Общая информация

Данный репозиторий содержит результаты выполнения учебного задания по Linux.

## Важная преамбула

Задание выполнялось на технически слабом ноутбуке, поэтому в качестве рабочей среды была использована облачная виртуальная машина в Yandex Cloud.

Изначально по условиям задания предполагалось использование Fedora 43. Однако в доступной облачной среде практическое выполнение на Fedora 43 через custom image сопровождалось инфраструктурными проблемами, не относящимися к сути самого задания (в частности, вопросы инициализации cloud-init, metadata и SSH-доступа). По этой причине для завершения практической части был использован стабильный и работоспособный публичный образ Fedora 37.

При этом все ключевые команды, настройки и административные действия, предусмотренные заданием, были выполнены в полном объеме: настройка сети, установка пакетов, работа с дисками и файловой системой, настройка nginx, SELinux, пользователей, прав доступа, capabilities, ограничений root-доступа, а также подготовка подтверждающих артефактов и публикация результатов в публичном GitHub-репозитории.

Таким образом, несмотря на отличие версии ОС от первоначально заявленной в формулировке задания, содержательная и практическая часть работы выполнена корректно.

## Использованная среда

- Облачная платформа: Yandex Cloud
- Виртуальная машина: `fedora37-mephi`
- ОС: Fedora 37
- Пользователь для работы: `yc-user`

## Выполненные пункты задания

### 1. Базовая система
- Развернута headless/minimal-среда Fedora на облачной VM.

### 2. Настройка сети
Настроены параметры сети:
- IPv4 address: `192.168.1.100/24`
- Gateway: `192.168.1.1`
- DNS: `8.8.8.8`
- Hostname: `mephi-2026.domain.local`

Результаты проверки сети сохранены в файл:
- `artifacts/network_check.txt`

### 3. Установка пакетов
Установлены требуемые пакеты:
- `nginx`
- `tcpdump`
- `libcap-ng-utils`

Подтверждение установки пакетов:
- `artifacts/package_check.txt`

### 4. Установка tcpdump через RPM
Пакет `tcpdump` был отдельно скачан через `dnf download`, после чего установлен через `rpm`, как требовалось по заданию.

### 5. Подготовка второго диска
Для дополнительного диска был выполнен полный цикл настройки:
- создан раздел на весь диск;
- создана файловая система `ext4`;
- задан label: `MEPHI_DATA`;
- настроено постоянное монтирование через `/etc/fstab`.

Точка монтирования:
- `/data/mephi-web`

Подтверждение:
- `artifacts/storage_check.txt`

### 6. Настройка nginx
Выполнено:
- запуск сервиса `nginx`;
- включение автозапуска;
- настройка размещения веб-страницы;
- проверка локальной выдачи страницы.

Текст страницы:
- `Hello from Student: М2551072`

Подтверждения:
- `artifacts/nginx_journal_last5min.txt`
- `artifacts/web_page_check.txt`
- `artifacts/security_service_check.txt`

### 7. Пользователи и группы
Созданы:
- пользователь `mephi-admin`
- группа `mephi-devs`

Для каталога `/data/mephi-web` настроены:
- групповая принадлежность;
- права `2775`.

Подтверждение:
- `artifacts/access_check.txt`

### 8. SELinux
SELinux работает в режиме:
- `Enforcing`

Для каталога `/data/mephi-web` назначен корректный SELinux context для работы с `nginx`.

Подтверждение:
- `artifacts/security_service_check.txt`

### 9. Capabilities для tcpdump
Для бинарного файла `tcpdump` настроены capabilities:
- `cap_net_raw`
- `cap_net_admin`

Подтверждение:
- `artifacts/security_service_check.txt`

### 10. Ограничение root-доступа
Настроены ограничения:
- запрещен вход `root` по SSH;
- локальный вход `root` заблокирован.

Подтверждение:
- `artifacts/access_check.txt`

### 11. Git и публикация результатов
- установлен `git`;
- создан локальный репозиторий;
- подготовлен и опубликован публичный GitHub-репозиторий с результатами работы.

Подтверждение:
- `artifacts/git_check.txt`
- `REPOSITORY_URL.txt`
